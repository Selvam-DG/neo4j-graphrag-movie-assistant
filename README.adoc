This repository contains a personal GraphRAG project built with Neo4j and Python.

The project demonstrates how Neo4j can be used with large language models to retrieve graph-based context and generate grounded responses.

The project includes examples of:

* Vector retrieval
* Vector-based GraphRAG
* Vector and Cypher retrieval
* Text-to-Cypher generation
* Neo4j GraphRAG pipelines
* Environment validation
* Automated testing

== Features

=== Vector Retriever

Uses a Neo4j vector index to identify semantically relevant nodes based on a natural-language query.

=== Vector GraphRAG

Combines Neo4j vector retrieval with a large language model to generate answers grounded in retrieved graph data.

=== Vector and Cypher GraphRAG

Uses vector search together with Cypher queries to retrieve semantically relevant content and connected graph information.

=== Text-to-Cypher GraphRAG

Converts natural-language questions into Cypher queries and uses the query results as context for an LLM-generated response.

== Setup

=== Clone the Repository

Clone the repository and enter the project directory.

[source,sh]

git clone https://github.com/Selvam-DG/neo4j-graphrag-movie-assistant.git
cd neo4j-graphrag-movie-assistant


=== Create a Virtual Environment

Create a Python virtual environment.

[source,sh]

python -m venv .venv
source .venv/bin/activate


On Windows PowerShell, use:

[source,powershell]

source .venv\Scripts\Activate.ps1

=== Install Dependencies

Install the required Python packages.

[source,sh]

pip install --upgrade pip
pip install -r requirements.txt


=== Configure Environment Variables

Create a `.env` file using `.env.example` as a template.

[source,sh]

 cp .env.example .env

Add your Neo4j and LLM credentials to `.env`.

[source,env]

NEO4J_URI=neo4j+s://your-instance.databases.neo4j.io
NEO4J_USERNAME=neo4j
NEO4J_PASSWORD=your-password
OPENAI_API_KEY=your-api-key


Do not commit the `.env` file.

== Environment Check

Run the environment validation script.

[source,sh]

python graphrag_app/test_environment.py

The script verifies that:

* Required environment variables are available
* The Neo4j database can be reached
* Required Python packages are installed
* The configured LLM provider can be accessed

== Security

Credentials are loaded from environment variables.

Never commit:

* `.env` files
* API keys
* Neo4j passwords
* Database credentials
* Private certificates
* Access tokens

Confirm that `.env` is not tracked before committing.

[source,sh]

git ls-files | grep -E '(^|/).env$'

The command should return no output.
